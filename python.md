## Data Types

### NoneType

**NoneType** is the type for the none object, which is an object that contains no value or defines a null value. It is used to indicate that a variable or expression does not have a value or has an undefined value.

"None" basically means the absence of a value.

**Syntax:**
```python
a = None
```

---

### List

Lists are used to store multiple items in a single variable.

**Characteristics:**
- List items are ordered, changeable, and allow duplicate value
- List index starts with 0
- If you add new items to a list, the new items will be placed at the end of the list
- The list is changeable, meaning that we can change, add, and remove items in a list after it has been created
- Using the `list()` constructor we can make a list

**Example:**
```python
thislist = list(("a","b","c"))
print(thislist)
>>> ['a', 'b', 'c']
```

---

### List Comprehension

List comprehension offers a shorter syntax when you want to create a new list based on the values of an existing list.

**Syntax:**
```python
newlist = [expression for item in iterable if condition == True]
```

**Condition:** The condition is like a filter that only accepts the items that evaluate to True

---

### Tuple

A tuple is an immutable ordered collection of elements. Once created, you cannot change, add, or remove elements. A tuple is a collection which is ordered and unchangeable and allows duplicate values.

**Key Points:**
- One cannot add items to a tuple once it is created
- Tuples cannot be appended or extended
- We cannot remove items from a tuple once it is created
- We can access elements from tuple using positive & negative indexing
- In case you're generating a tuple with a single element, make sure to add a comma after the element

**Syntax for single element:**
```python
tup = ("a",)
```

**Available Functions:** `len()`

**Operations on Tuple:**

1. **Concatenation** - Use plus operators to concatenate tuples
   ```python
   tuple1 + tuple2
   ```

2. **Nested Tuple** - We can create tuple inside tuple
   ```python
   tup3 = (tup1 + tup2)
   ```

3. **Slicing** - We can divide tuples into smaller tuples
   ```python
   print(tup[::−1])
   ```

4. **del** - It will delete tuple. Post that if we try to access that tuple it will give NameError

---

### Ways to Edit Tuple

1. **Convert to list, modify, convert back to tuple:**
   - Use `list()` function to convert the tuple into a list
   - Modify the list (change values, add or remove elements)
   - Use `tuple()` function to convert the list back into a tuple

2. **Create a new tuple:**
   - Use the `+` operator to join the old tuple with new values
   - Specify which part of the original tuple you want to keep using index positions
   - The result is completely new tuple with your changes

3. **Use slicing & Concatenation:**
   - Use slicing notation (like `[:2]` or `[3:]`) to split the tuple into parts
   - Join the parts together using `+` operator with new values in the middle
   - The result combines the untouched parts with your new values

4. **Use unpacking:**
   - Unpack the tuple into individual variables using commas
   - Modify the variables you want to change
   - Pack them back into a new tuple using parentheses
   ```python
   tup = ("a","b")
   ```

### Range

The `range()` function returns a sequence of numbers, starting from 0 by default and increments by 1 and stops before a specified number.

**Syntax:**
```python
range(start, stop, step)
```

---

### Dictionary

Dictionaries are used to store data in key:value pairs. A dictionary is a collection which is ordered, changeable and does not allow duplicates.

**Key Points:**
- Values in a dictionary can be of any data type and can be duplicated, whereas keys can't be repeated and must be immutable
- One value at a time can be added to a dictionary by defining value along with the key: `dic[key]='value'`
- Updating an existing value in a dictionary can be done by using the built-in `update()` method
- While adding a value, if the key-value already exists, the value gets updated otherwise a new key with the value is added to the dictionary
- The KeyError exception you will see when a dictionary is accessed with either an undefined key or by a numeric index

---

### Dictionary Comprehension

Dictionary comprehension is a feature that allows you to create a dictionary from a list or another dictionary in a single line of code.

**Syntax:**
```python
squares = {x: x*x for x in range(6)}
```

### Set

Sets are used to store multiple items in a single variable.

**Characteristics:**
- A set is a collection which is unordered, unchangeable, unindexed
- Set is mutable, i.e. we can remove or add elements to it

**Syntax:**
```python

```

## Functions

### Map

Map function is used to apply given function to each item of an iterable (like a list or tuple) and returns an iterable (which can be converted into a list, tuple, etc)

**Syntax:**
```python
map(function, iterable)
```

**Parameters:**
- **Function** - A function to which the items of the iterable are passed
- **Iterable** - An iterable (like a list, tuple) whose elements are to be processed by the function

**Note:** You can also pass multiple iterables, the function should accept as many arguments as there are iterables.

---

### Filter

Filter function is used to filter elements from an iterable based on a condition defined by a function. The function tests each element of the iterable and returns an iterator that contains only those elements for which the function returns True.

**Syntax:**
```python
filter(function, iterable)
```

**Parameters:**
- **Function** - Function that tests whether each element of the iterable should be included in the result. This function should return a boolean value (true, false)
- **Iterable** - The iterable (list or tuple) whose elements want to filter

**Important:** The filter function returns an iterator. Like `map()` it does not immediately produce a list. If you want to see the filtered result, you need to convert it into tuple or list.

## Object-Oriented Programming (OOP)

### Class

Class is a blueprint for creating objects.

**Types:**
- **Abstract Class** - A class that cannot be instantiated directly and is meant to serve as a blueprint for other classes. Defined using the abc (Abstract base class) module. Contains abstract methods.
- **Static Class** - A class which only contains `@staticmethod`

---

### Object

An object is an instance of a class. It represents a specific entity that has the attributes and behaviors defined by its class. You can create multiple objects from the same class and each with its own unique state.

---

### Parameter

Parameters are the inputs to the function or method. They allow you to pass data into these callable entities. In the context of classes, parameters are often used in the constructor (`__init__`) to initialize object attributes.

---

### __init__ Method

The init method works similarly to the constructors in Java. The method is run as soon as an object is instantiated. It is useful for initializing any attributes or default behaviour of the object at the time of instantiation.

---

### Lambda

Lambda is a small, anonymous function that is defined using lambda keyword.

**Syntax:**
```python
lambda arguments: expression
```

**Example:**
```python
add = lambda x, y: x + y
print(add(3, 5))
```

---

### Polymorphism

Polymorphism means many forms. Polymorphism allows objects of different class to respond to the same method. In simple word, Polymorphism means that a single action or a single method can behave in different ways, depending on the context or the object involved. (Many forms, same method different behavior)

---

### Encapsulation

Encapsulation restricts direct access to some components of an object, ensuring that controlled interaction with the data and its security.

**Access Levels:**
- **Public member** - Accessible from anywhere
- **Protected members** - Indicated by a single underscore `_`. It's accessible within the class and its subclass
- **Private members** - Identified by double underscore `__`. It's accessible only within the class

---

### Inheritance

Inheritance allows one class to derive or inherit the properties and behaviors of another class.

---

### Inheritance Types

**Types of Inheritance in Python:**

1. **Single Inheritance:** A child class inherits from a single parent class
2. **Multiple Inheritance:** A child class inherits from multiple parent classes
3. **Multilevel Inheritance:** A class is derived from a child class, forming a chain
4. **Hierarchical Inheritance:** Multiple child classes inherit from a single parent class
5. **Hybrid Inheritance:** A combination of two or more types of inheritance

---

### Method Overriding vs Method Overloading

| Feature | Method Overriding | Method Overloading |
|---------|-------------------|-------------------|
| **Definition** | Same method, different class | Same method, different parameters |
| **Location** | Parent and child classes | Same class |
| **Parameters** | Parameters are same | Parameters are different (different number or types) |
| **Inheritance** | Inheritance is required | Inheritance is not required |
| **Purpose** | Child customization | Handle different inputs |

**Important Note:** Python does not support overloading - but we can use default parameters `*args`, `**kwargs`. Python is dynamically typed and resolves function calls at runtime. Defining multiple methods with the same name overrides the previous one.

---

### @staticmethod

A `@staticmethod` is a function inside a class that doesn't need to use self. It works like a normal function, but it's placed inside a class.

---

### Getter and Setter

- **Getter** - A method that gets the value of an attribute
- **Setter** - A method that sets or updates value of an attribute

It's mostly used in encapsulation to keep internal details of a class hidden.

---

### Aggregation

A relationship where one class contains another class, but both can exist independently.

**Example:** A car has wheels, but wheels can exist without a car. You own a phone, but phone exists independently. You can replace it, give it away, etc.

---

### Composition

A relationship where one class contains another class, and the contained class cannot exist independently.

**Example:** A car has an engine, but an engine cannot exist as a standalone car engine. Your body has organs. Organs cannot exist independently from you. When you die, organs also cease to exist.

---

### Aggregation vs Composition

| Feature | Aggregation | Composition |
|---------|-------------|-------------|
| **Part Independence** | Parts exist independently | Parts depend on whole |
| **Ownership** | Weak ownership | Strong ownership |
| **Survival** | Parts survive if whole is deleted | Parts are deleted with whole |

---

### Singleton Class

A singleton class is a class that can have only one object at a time. After the first time, if you try to instantiate the singleton class, the new variable also points to the first instance.

**Main use:** Control resource access - Ensure only one instance of a class exists throughout the entire program. Database connection, logger, configuration settings

---

### Super

Super is a built-in function that allows you to call methods from the parent (based) class inside the child (derived) class.

**Purpose:** Access parent class methods, properties or constructor without writing the parent class name explicitly.

---

### Overloading

Overloading is a feature that allows you to define multiple functions or methods with the same name but different parameters.

---

### Private Methods

A private method is a method that is intended to be used only inside its own class and not accessed directly from outside the class. It uses name mangling as a convention to indicate privacy.

**Key Points:**
- Use private methods when you want to hide internal implementation details and expose only the necessary interface to the outside world
- If a function starts with two underscores then it is private method: `__methodname`
- To access this private method we need to give 2 underscore to class name followed by 2 underscore to method name then we can access it: `__classname__methodname`
- It's not strictly private, you can still access it using the mangled name, but it's discouraged

---

### Access Modifiers

In python control which parts of a class can be accessed from outside the class, from within the class or by subclasses.

| Access Level | Public | Protected | Private |
|---|---|---|---|
| **Prefix** | No prefix needed | Single underscore prefix is needed | Double underscore prefix is needed |
| **Accessibility** | Accessible outside class (everyone can use) | Accessible outside class (discouraged) (Internal & subclasses) | Not accessible outside class (class only) |
| **Enforcement** | No enforcement | Convention enforcement | Name mangling enforcement |
| **Use Case** | API/normal usage | Internal logic | Sensitive data |

## Memory Management

### Memory Management in Python

Memory management in Python is handled by the Python Memory Manager. The memory allocated by the manager is in form of a private heap space dedicated to Python. All Python objects are stored in this heap and being private, it is inaccessible to the programmer. Though, python does provide some core API functions to work upon the private heap space.

Additionally, Python has an in-built garbage collection to recycle the unused memory for the private heap space.

---

### Generators

Generators are functions that return an iterable collection of items, one at a time, in a set manner. Generators, in general, are used to create iterators with a different approach. They employ the use of yield keyword rather than return to return a generator object.

**Main use:** Save memory - Process large amounts of data without storing everything in memory at the same time.

---

### Garbage Collection

Garbage collection is like a cleaning service for your computer's memory. When your program runs, it uses memory to store things like variables, objects, and data. But once you're done using them, they don't automatically disappear. Garbage collection is the process where Python automatically cleans up and frees the memory that is no longer needed.

**How Does It Work?**

1. **Reference Counting:**
   - Every time you create an object in Python (like a list or a number), Python keeps track of how many times other parts of your code are using that object. This is called reference counting
   - If no part of your code is using the object anymore, Python knows it can safely remove it and free up memory

2. **Cyclic Garbage Collection:**
   - Sometimes, objects can refer to each other, creating a "cycle" of references. For example, if two objects point to each other, their reference counts might never reach zero, even if they aren't needed anymore. This can cause memory to be used up unnecessarily
   - Python has a special garbage collector that looks for these cycles and removes them to free up memory

## Concurrency

### Multithreading

Multiple threads run within a single process, sharing the same memory space.

**How it works:** Threads take turns executing due to python's GIL (Global interpreter lock)

**GIL Impact:** Only one thread executes python bytecode at a time, so it does not achieve true parallelism for CPU-bound tasks.

**Best for:** I/O bound tasks - file reading, network requests, database queries, API calls

**Analogy:** One chef (CPU) switching between multiple dishes (threads) - only cooking one at a time but switching quickly during wait times

**Pros:**
- Lightweight (lower memory overhead)
- Shared memory makes communication easy
- Great for waiting on external resources

**Cons:**
- GIL prevents true parallel CPU execution
- Race conditions & deadlock possible
- Needs locks/semaphores for thread safety

---

### Thread

Thread is a pre-defined class which is available in threading module. To use thread, we need to import it.

---

### Global Interpreter Lock

Global Interpreter Lock (GIL) is used to manage the execution of only one thread at a time.

**Important Note:** Python supports multithreading via a threading module. But due to the GIL python threads can't fully utilize multiple CPU cores for CPU bound task.

---

### Deadlock

A deadlock is a situation where two or more threads or processes are unable to process because each is waiting for the other to release a resource (lock).

---

### Async

Async is a program to execute multiple tasks concurrently without waiting for each task to complete before starting the next one.

It is a single-threaded, single-process concurrency using an event loop and cooperative multitasking. Tasks voluntarily yield control back to the event loop, which then runs other tasks. No preemption - tasks must cooperate.

**Best for:** High volume I/O bound tasks - handling thousands of simultaneous network connections, web scraping, chat servers, API gateways

**Analogy:** One chef with a perfect timer system - starts boiling water, while waiting sets up the oven, while waiting chops vegetables - never idle, but still one person

**Pros:**
- Extremely lightweight
- No thread-switching overhead
- No race conditions
- Very efficient for massive I/O concurrency

**Cons:**
- Cannot help with CPU-bound tasks
- One blocking call freeze everything
- Requires async/await throughout the codebase
- Harder to debug sometimes

---

### Asyncio vs Multithreading

| Feature | Asyncio | Multithreading |
|---------|---------|-----------------|
| **Model** | Cooperative (Single-thread) | Preemptive (Multiple threads) |
| **Execution** | Execution happens via event loop | Execution happened via thread-based |
| **Analogy** | Single chef incredibly fast and organized. They start cooking pasta, then while it's boiling they quickly chop vegetables for a salad. They constantly switch between tasks, making progress on each whenever there is a pause in another | Multiple chefs in the kitchen, each with their own stove workspace. One chef can prepare pasta, and another make salads – all at the same time. But they need to be careful not to bump into each other or use some tools at the same time |
| **Example** | Web server that needs to handle many client requests. With Asyncio, a single thread can handle multiple requests by switching between them whenever a request needs to wait for something | Program that needs to download multiple files from the internet. With multithreading, you can have each thread download a separate file simultaneously, speeding up the overall process |

---

### Multiprocessing

Multiple separate processes, each with its own memory space and own GIL.

**How it works:** OS schedules processes across multiple CPU cores achieving true parallelism

**Best for:** CPU-bound task - mathematical computations, image processing, data crunching, ML training

**Analogy:** Multiple chefs (CPU) each with their own kitchen (memory) - truly cooking different dishes simultaneously

**Pros:**
- True parallel execution on multiple cores
- No GIL limitation
- One process crashing doesn't kill others

**Cons:**
- Heavy memory usage
- Inter-process communication (IPC) is complex and slow
- Higher startup cost

---

### Multithreading vs Multiprocessing vs Async

| Feature | Multithreading | Multiprocessing | Async |
|---------|---|---|---|
| **Execution** | Concurrent (not parallel due to GIL) | Execution is true parallel | Execution concurrent |
| **Processes/Threads** | Threads are multiple, 1 process | Multiple processes | One thread, one process |
| **Memory** | Memory is shared | Memory is separate per process | Memory is shared |
| **Best For** | I/O bound | CPU-bound task | High-volume I/O bound |
| **Scalability** | Hundreds of threads | Limited by CPU cores | Thousands of coroutines |
| **GIL** | GIL is affected | GIL is not affected | GIL is affected but irrelevant |
| **Complexity** | Medium | High | Medium |
| **Risks** | Race conditions and deadlocks | Memory overhead and slow IPC | Blocking calls freeze everything |

### Semaphore

A semaphore is not a concurrency model itself. It a synchronization primitive used within any of the multiprocessing, multithreading, async.

It limits the number of concurrent accesses to a shared resource.

**How it works:**
- It maintains an internal counter
- Each acquisition decrements it, each release increments it
- When counter reaches zero, further acquisitions block/wait

**Analogy:** A parking lot with limited spots - only N cars allowed in at a time. Others must wait until someone leaves.

---

### Race Condition

A race condition occurs when two or more threads or processes access shared data at the same time, and the final outcome depends on the timing or order of execution. This can lead to unpredictable and incorrect results.

**Why it happens:**
1. Python threads share the same memory space
2. If multiple threads read and write to the same variable without proper synchronization, the operations can interleave in unexpected ways
3. Even though python has the Global Interpreter Lock (GIL), it does not prevent race conditions for operations that are not atomic

**Important Points:**
- Race conditions occur when shared mutable state is accessed without synchronization
- The GIL does not protect against them for non-atomic operations
- Use locks, semaphores, queues or other synchronization primitives to avoid them

---

### Mutex

Mutex is a synchronization primitive used to prevent race conditions in concurrent programming.

## Language Concepts

### Interpreted Language

Python is called an interpreted language because its code is executed line by line by the python interpreter at runtime without a separate compilation step.

**How it works:**
1. Python source code (.py) is written by you
2. The python interpreter converts it into bytecode (.pyc)
3. This bytecode is executed immediately by the python virtual machine (PVM)

**Important Note:** Python is both a compiled and an interpreted language because its source code is first compiled into bytecode, and then this bytecode is interpreted and executed by the python virtual machine at runtime.

## Operators

### Ternary Operator

The if else condition has only true and false then use ternary operator.

**Syntax:**
```python
<exp1> if <condition> else <exp2>
```

**How it works:**
1. First evaluates `<condition>`
2. If it is True, evaluates and returns `<exp1>` but does not evaluate `<exp2>`
3. If it is False, evaluates and returns `<exp2>` but does not evaluate `<exp1>`

## Security

### OAuth

OAuth is a standard designed to allow a website or application to access resource hosted by other web apps on behalf of a user. OAuth 2.0 provides consented access and restricts actions of what the client app can perform on resources on behalf of the user, without ever sharing the user's credentials.

**Key Points:**
- It uses access tokens to grant access to resources on behalf of the end-user
- These tokens may have an expiration date for security reasons

**Components:**
- **Resource Owner:** The user or system that owns the protected resources and can grant access to them
- **Client:** The system that requires access to the protected resources. It must hold the appropriate access token
- **Authorization Server:** Issues access tokens upon successful authentication and consent by the resource owner
- **Resource Server:** Protects the user's resources and validates access tokens from the client

## Web Framework - FastAPI

### FastAPI

FastAPI is the fastest web framework available, largely due to its asynchronous capabilities and optimized code generation.

**Features:**
- It's the fastest web framework available
- It generates interactive documentation using Swagger UI or Postman
- Automatic data validation and serialization using pydantic library

---

### Pydantic in FastAPI

Pydantic is used in FastAPI for data validation and settings management.

**Key Points:**
- It helps incoming request data, query parameters
- It automatically serializes python objects to JSON
- It ensures that data matches the expected structure and types

---

### Route in FastAPI

Route is a function that handles HTTP requests for specific URL path and HTTP method.

---

### How is the route defined in FastAPI?

Routes in FastAPI are defined using decorators such as `@app.get()`, `@app.post()`, `@app.put()`

---

### Async and Await in FastAPI

In FastAPI async and await are used for asynchronous programming.

**Why they are important:**
- By using `async def` in route handlers, you can perform non-blocking I/O operations
- Essential for handling multiple requests concurrently
- Improves the performance of the API
- Important when you need to handle many requests simultaneously, especially for tasks that query database, call external API or read/write to disk

---

### Request and Response Objects

**Request** – Used to access incoming request data, such as headers, query parameters and body content. It can injected into your route functions.

**Response** – Used to customize outgoing response, such as setting headers, status codes, and returning content in custom format.

---

### Error Handling in FastAPI

FastAPI provides build-in support for handling errors using HTTPException. You can raise this exception with a custom status code and detail message.

---

### Schema in FastAPI

In FastAPI schema refers to pydantic models that define the structure of request and response bodies.

**Purpose:**
- These models validate data types
- Provide automatic documentation

---

### CORS in FastAPI

CORS (Cross-Origin Resource Sharing) helps to enable requests from different domains.

**How to enable CORS:** FastAPI provides easy integration for CORS through the CORSMiddleware. This middleware allows you to specify which domains or methods are allowed to access your API.

---

### BaseModel in FastAPI

BaseModel is the base class provided by pydantic for definition data models.

**Purpose:**
- These models help to define request bodies, query parameters and other data structures in FastAPI routes
- It provides automatic validation and serialization

### Flask-WTF

Flask-WTF (WTF also known as WTForms in flask), is a type of interactive user interface.

**Features:**
- Integrate with web forms
- Support captcha

---

### Database Connections in Flask

Creating and closing database connections all the time is very inefficient, because database connections are encapsulated a transaction, you must ensure that the connection is only used by one request at a time.

**Three ways to request database:**

1. **before_request()** – No parameters are given when these connections are invoked before making a request

2. **after_request()** – After initiating a request these connections are called, and a response is sent to the client

3. **teardown_request()** – This decorator is called when an exception is raised or everything went well

---

### Thread-local Object in Flask

Flask Python uses thread-local objects so that the user does not have to transmit objects from one function to the next within a request to stay thread safe.

## Version Control - Git

### Git

Git is a version control system. It tracks changes in your code/files, allows you to go back to previous versions, and enables collaboration with other developers.

---

### VCS (Version Control System)

A VCS is a tool that keeps record of file changes.

**Purpose:**
- Shows who changes what and when
- Helps avoid overwriting other works

---

### .gitignore

.gitignore is a file where you list files or folders git should ignore.

**Common items to ignore:**
- Log files
- Temporary files
- Keys

---

### Repository

A repository is your project folder that git is tracking.

**Types:**
- **Local repo:** Your laptop
- **Remote repo:** Your GitHub

---

### Git Add

Git add says "I want to include these changes in the next commit." It moves changes to staging area.

---

### Git Status

Git status shows which files changed, which are staged, which are untracked.

---

### Git Commit

A commit saves your staged changes with a message. Think of commit as a checkpoint or snapshot.

**Syntax:**
```bash
git commit -m "added login feature"
```

---

### Git Push

Git push sends your local commits to the remote repository.

---

### Git Pull

Downloads changes from remote. Merges them into your branch.

**Syntax:**
```bash
git pull origin main
```

---

### Git Clone

Git clone creates a copy of a remote repository on your machine. First step when starting work on an existing project.

**Syntax:**
```bash
git clone https://...
```

---

### Git Fetch

Downloads changes from remote, does not merge them. Safe way to see updates without affecting your code.

---

### Git Checkout

Used to switch branches, restore files, move to another branch.

**Syntax:**
```bash
git checkout -b new-branch
```

---

### Conflict

A conflict happens when two people change the same lines. Git can't decide which one to keep. You must manually fix it.

---

### Git Stash

Git stash temporarily saves your uncommitted changes. Use it when you need to switch branches quickly or your code isn't ready to commit.

---

### Git Apply

Brings back stashed changes. Restores your work.

---

### Git Reset

Git reset moves HEAD to a previous commit.

**Types:**
- **Soft** – Keeps changes staged
- **Mixed** – Keeps file changes
- **Hard** – Deletes changes

**Syntax:**
```bash
git reset –hard commit_id
```

---

### Git Revert

Creates a new commit that safely undoes a previous commit. Best for shared branches (safe undo).

**Syntax:**
```bash
git revert commit_id
```
